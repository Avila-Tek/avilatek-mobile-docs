# Reglamento de Estilo de Código del equipo móvil

El **Reglamento de Estilo de Código del equipo móvil** (RECm) son un conjunto de reglas que se utilizan para garantizar la coherencia en el código base de nuestros proyectos.
Estas reglas deben seguirse cuando sea posible para mantener un código base coherente y legible.

El RECm está altamente opinionado en torno a la [Arquitectura Limpia](https://blog.cleancoder.com/uncle-bob/2012/08/13/the-clean-architecture.html) y los [Principios de Limpieza de Código](https://g.co/kgs/qunvt4A), la arquitectura seguida por el equipo.


:::info[agregar nuevas reglas]


Para agregar una nueva regla, lee la [guía de contribución de al Reglamento de Estilo de Código](https://github.com/Avila-Tek/avilatek-mobile-docs/blob/main/CONTRIBUTING.md).

:::

## Secciones del RECm

1. General guidelines: Naming conventions, Formatting, Commenting
2. Data layer guidelines
    1. Data sources
    2. Models
    3. Repository implementations
3. Domain layer guidelines
    1. Use cases
    2. Entities
    3. Repositories
4. Presentation layer guidelines
    1. Bloc
    2. Page-view-body components 
    3. Widget atomic components
5. Testing guidelines
6. Other guidelines
